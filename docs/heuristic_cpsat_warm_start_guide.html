Heuristic Warm Start for CP-SAT

This document explains the relationship between the custom greedy scheduler and CP-SAT: the heuristic finds a fast, valid incumbent first, then CP-SAT uses that schedule as a warm start and searches for a better global minimum under the same model.

Greedy first CP-SAT hinting Global search Feasible incumbent Bound + gap
Core mental model: the heuristic does not compete with CP-SAT. It gives CP-SAT a head start so the solver can spend its time improving a real schedule instead of discovering the first valid one from scratch.
Important: CP-SAT objective quality and dashboard metrics are related but not identical. A solver-improved objective can still look worse on one descriptive metric such as max individual wait.

One-Line Flow

The warm start is only useful if the heuristic schedule can be translated into a CP-SAT assignment for each bus, station, lane, and start time.

What Each Solver Is Doing

Custom heuristic

Builds a schedule bus by bus using local decisions. It is fast, deterministic, and usually finds a good-feeling schedule quickly.

CP-SAT

Treats the full schedule as a constraint model and searches the solution space for a better global arrangement under the same rules.

Warm start

Gives CP-SAT a known-valid incumbent. That means the search begins with a real schedule already in hand instead of a blank slate.

Heuristic-to-CP-SAT Mapping

Heuristic artifact CP-SAT variable Why it matters
Chosen station sequence candidate.selected = 1 Locks in the same route pattern as the warm start.
Charge start time stop.start = heuristic_start Seeds the solver with a complete, time-feasible schedule.
Charger lane lane_selected = 1 Preserves the exact physical charger assignment.
Why the mapping must be exact: CP-SAT only benefits from hints if they line up with its own candidate structure. If a bus’s heuristic route cannot be represented as a CP-SAT candidate, the hint should be skipped rather than partially applied.

Decision Tree

How CP-SAT Searches For A Global Minimum

1. Start with a feasible incumbent

The heuristic schedule is already valid, so CP-SAT can begin with an objective value instead of zero knowledge. That gives the solver a target to beat.

2. Propagate constraints

CP-SAT tightens variable domains based on no-overlap, route order, charging duration, and range constraints.

3. Branch when necessary

When multiple values remain possible, the solver splits into search branches. It tries alternatives, keeps the best feasible schedules, and prunes options that cannot beat the current incumbent.

4. Use bounds to measure progress

The best objective bound tells you how far the solver can mathematically prove it is from the true optimum.

Read the search like a story: CP-SAT is not marching directly downhill. It is exploring a huge tree of possibilities, keeping the best incumbent it has seen, and using bounds to prove when it can stop.

How To Interpret Diagnostics

Signal What it usually means
FEASIBLE + nonzero gap A valid schedule was found, but the proof is incomplete.
Objective Improvement > 0 CP-SAT beat the heuristic incumbent on the solver objective.
Best Objective Bound = 0 The solver did not establish a useful lower bound during the time limit.
High conflict and branch counts The search worked hard, but may not have found much extra value.

Why The Dashboard Can Feel Counterintuitive

The UI shows both descriptive schedule metrics and solver diagnostics. A CP-SAT solution can look worse on one human-facing metric and still improve the solver objective because the objective function is a weighted blend. 

solver objective = scaled_wait + scaled_journey_time
Human translation: if you care mainly about one metric such as max wait, check whether the objective function is actually aligned with that preference.

Quick Summary

Heuristic Finds the first good valid answer quickly.
Hint Hands that answer to CP-SAT as a starting incumbent.
Search Explores alternative assignments to lower the objective.
Diagnostics Tell you whether CP-SAT improved, stalled, or proved optimality.
