ExpoNow Test Infrastructure

A complete digest of what we verify, how, and why. 15 test files covering 11 source modules with 6 fixtures and 5 assignment scenarios. All tests run via python -m unittest discover — no pytest, no CI/CD, no coverage tooling. Python 3.11 (runtime.txt). OR-Tools is optional; CP-SAT tests skip gracefully.

Recurring Test Patterns

Canonical E2E Pipeline

Used in 5+ test files. Load scenario, schedule, validate invariants.

scenario = load_scenario("scenario_1")
result = CustomHeuristicStrategy().schedule(scenario)
violations = validate_schedule_invariants(scenario, result)
self.assertTrue(result.feasible)
self.assertEqual(len(violations), 0)

validate_schedule_invariants & None scenario

Deliberate seam: early-exit for infeasible results accepts None scenario.

result = ScheduleResult(
    feasible=False,
    scenario_id="test")
violations = validate_schedule_invariants(
    None, result)  # type: ignore
self.assertTrue(len(violations) > 0)

Invoking the Tests

Primary command 
python -m unittest discover
This discovers all test files in tests/ (a flat package) and runs them. No configuration files are needed — the discovery works by default because tests/__init__.py makes it a Python package.
Run a single file 
python -m unittest tests/test_file.py
Useful during development for quick feedback on a specific module.
Environment notes
  • Python 3.11 (specified in runtime.txt)
  • Test runner: vanilla unittest — no pytest, no tox, no executable test scripts
  • No coverage tooling installed (no .coveragerc, no coverage.py)
  • OR-Tools is an optional dependency; CP-SAT tests skip gracefully via @unittest.skipUnless
  • No CI/CD pipeline — relies on manual invocation or developer discipline

Gap Analysis

Deliberate or accepted gaps in the current test infrastructure. None are framed as criticism — all reflect conscious design decisions or future opportunities.

Missing Severity Note
CI/CD pipeline Medium No automated test gate; relies on manual python -m unittest discover
Code coverage measurement Low No coverage.py or threshold enforcement
Pytest adoption Low unittest is fine for this scale; pytest would give nicer output and plugins
Performance / benchmark tests Low No timing assertions; scheduler engine timing is untracked
UI / Streamlit tests Low No browser automation; architecture guards instead
Design philosophy The architecture boundary guards (asserting no Streamlit imports in src/scheduler/ and src/reporting/) and the AST-based import check in test_increment_5 represent a conscious tradeoff: verify isolation statically rather than spinning up a browser. This keeps the test suite fast and dependency-light while still enforcing the layered architecture.
